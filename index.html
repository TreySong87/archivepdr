GGProduction 아카이빙 시스템 구축 PDR 상세 보기

GGProduction 아카이빙 시스템 PDR

목적: 본 문서는 GGProduction 아카이빙 시스템 구축을 위한 핵심 필수 요구사항을 정의합니다.

1. 프로젝트 개요

  • 고성능 검색: 자연어 검색, 자동완성 등 출중한 검색 엔진 확보.
  • 프로세스 자동화: AI 기반 자동 서브클리핑 및 태깅.
  • AI 확장성: '포커' 도메인 최적화 및 내부 AI 엔진 연동.
  • 사용 편의성: 비전문가용 UI 및 워크플로우 커스터마이징.

1.2. 비-목표 (범위 한정)

  • 클라우드 전용(Upload Only) SaaS 솔루션 제외.
  • 물리적 테이프(SAMMA) 변환 제외.
  • 단순 파일 저장소(Drive) 및 LTO 제외.

2. 핵심 필수 요구사항 상세

2.1. (핵심 1) 출중한 검색 엔진 자연어 시맨틱 검색("Hellmuth가 화내는 장면"), 자동 완성, 연관 검색어 제공.
2.2. (핵심 2) 자동 Subclip과 Tagging AI 영상 분석을 통한 주요 장면(올인, 쇼다운) 인식 및 메타데이터(얼굴, 음성, OCR) 자동 생성.
2.3. (핵심 3) 포커 AI 최적화 및 확장 [옵션 A] 벤더 AI 커스터마이징(추가 학습) 또는 [옵션 B] 내부 AI 분석 결과 Write-back API 제공.
2.4. (핵심 4) 최적화된 UI/UX 비전문가도 즉시 사용 가능한 UI 및 워크플로우 커스터마이징 지원.

2 솔루션 선정 지연 사유 (The Blocker)

왜 선정이 늦어졌는가? : 시장의 기술적 한계와 검증 실패

Semantic Search 부재

대다수 MAM 솔루션이 단순 '텍스트 매칭' 수준이며, 우리가 요구하는 "상황 맥락 검색(Context Search)" 구현에 실패함.

Customization 검증 난항

여러 유력 후보군과 PoC를 진행했으나, '포커' 특화 데이터 구조(핸드, 팟 등) 커스터마이징 실패 또는 과도한 개발 기간 소요.

➡ 결론: 범용 방송용 솔루션의 한계 확인. AI 검색 특화 및 API 확장성이 검증된 솔루션으로 후보 재선정.

3 최종 후보 솔루션 비교

상세 비교표 보기

A. Moments Lab

(구 Newsbridge)

PDR 1.1(검색) & 1.2(자동화) 특화

  • 강점: 멀티모달 AI 엔진 탑재로 별도 학습 없이 자연어 검색 성능 우수.
  • 적합성: 방대한 아카이브에서 '원하는 장면을 즉시 찾아내는 것'이 최우선일 때 강력.

B. Dalet Flex

Enterprise Workflow

PDR 1.3(확장성) & 워크플로우 관리 특화

  • 강점: 엔터프라이즈급 엔진으로 타 시스템(OTT, 내부 AI) 연동 및 배포 파이프라인 설계 유리.
  • 적합성: 시스템의 '안정성, 확장성, 커스터마이징'이 최우선일 때 표준.

4 기술팀 검토 요청 사항 (RFP Requirements)

최종 결정을 위해 다음 4가지는 기술팀의 면밀한 검증(Feasibility Check)이 필요합니다.

Point 1

Data Migration (from Iconik)

현재 사용 중인 Iconik의 메타데이터와 에셋을 손실 없이 이관할 수 있는가? (매핑 호환성 점검)

Point 2

API & Integration (Poker AI)

자체 구축할 'Poker AI'와 연동하여, 분석 데이터를 기반으로 자동 Subclipping 및 Auto Tagging을 트리거(Trigger) 할 수 있는가?

Point 3

Security Architecture

외부 편집자 협업 시 원본 유출 방지(Proxy 워크플로우) 및 권한 제어(ACL) 정밀도.

Point 4

Future Scalability (OTT)

추후 구축될 자사 OTT 시스템 및 타 플랫폼과의 기술적 연동(Publishing Pipeline)이 원활한가?

결론 및 향후 계획
